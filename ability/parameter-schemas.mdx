Define the structure and validation for your Ability's input parameters using Zod schemas.

## abilityParamsSchema

The `abilityParamsSchema` defines what parameters executors must provide to your Ability. Include parameters for both your core functionality and any supported Policies.

```typescript
import { z } from 'zod';

const abilityParamsSchema = z.object({
  // Core ability parameters
  tokenAddress: z.string().regex(/^0x[a-fA-F0-9]{40}$/, "Invalid Ethereum address"),
  amountToSend: z.number().positive("Amount must be positive"),
  recipientAddress: z.string().regex(/^0x[a-fA-F0-9]{40}$/, "Invalid recipient address"),

  // Optional parameters
  gasLimit: z.number().optional(),
  customData: z.string().optional(),
});
```

## Advanced Schema Features

### Conditional Validation

```typescript
const abilityParamsSchema = z.object({
  tokenAddress: z.string(),
  amountToSend: z.number(),
  recipientAddress: z.string(),
  useCustomGas: z.boolean().optional(),
  gasLimit: z.number().optional(),
}).refine((data) => {
  // If useCustomGas is true, gasLimit is required
  if (data.useCustomGas && !data.gasLimit) {
    return false;
  }
  return true;
}, {
  message: "gasLimit is required when useCustomGas is true",
  path: ["gasLimit"],
});
```

### Transform Values

```typescript
const abilityParamsSchema = z.object({
  tokenAddress: z.string().transform((val) => val.toLowerCase()),
  amountToSend: z.string().transform((val) => parseFloat(val)),
});
```

## Integration with Policies

When supporting Policies, ensure your schema includes their required parameters:

```typescript
// Your Ability needs these for core functionality
const coreParams = {
  tokenAddress: z.string(),
  recipientAddress: z.string(),
  amountToSend: z.number(),
};

// Spending limit policy needs these additional params
const policyParams = {
  // tokenAddress: already included above
  // amount: maps to amountToSend above
};

const abilityParamsSchema = z.object({
  ...coreParams,
  // Policy params are handled via parameter mappings
});
```

## Type Inference

Get TypeScript types from your schemas:

```typescript
type AbilityParams = z.infer<typeof abilityParamsSchema>;
// Results in:
// {
//   tokenAddress: string;
//   amountToSend: number;
//   recipientAddress: string;
//   gasLimit?: number;
//   customData?: string;
// }
```

## Error Handling

Zod provides detailed validation errors:

```typescript
const result = abilityParamsSchema.safeParse(userInput);

if (!result.success) {
  // result.error.issues contains detailed error information
  console.log("Validation errors:", result.error.issues);
}
```
